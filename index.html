Module 01 · Foundations

The five pillars we will defend

Every control, policy, and instinct you build today maps to one of these five pillars. Memorise them — they are the lens you'll use the rest of the day.

C
Confidentiality
Customer data, account balances, transaction history — visible only to those authorised
I
Integrity
Transactions, ledgers, and records cannot be altered without trace
A
Availability
ATMs, online banking, and SWIFT must stay operational under attack
N
Non-Repudiation
Customers cannot deny actions; bank cannot deny obligations — digital signatures, audit logs
Z
Zero Trust
Verify every user, every device, every session — internal network is no longer "safe"

A bank is three things at once

A bank is simultaneously a vault, a switchboard, and a database. Attackers target whichever is weakest. Your job is to make sure none of them are weak today.

Module 03 · Reference

The SLAM method

Before clicking, hover, or replying to any email — run the SLAM check. Four seconds. Four letters. Saves millions.

S
Sender
Hover the name. Is the actual address the real domain? paypal-secure.com ≠ paypal.com
L
Links
Hover every link before clicking. Mismatched display text and URL = phishing.
A
Attachments
Unexpected file? Especially .exe .iso .lnk .zip .docm .xlsm — do not open. Confirm out-of-band.
M
Message
Urgency, threats, money, secrecy, grammar errors, generic greetings — any one = red flag.

The seven deadly tells

!
Urgency — "in the next 2 hours", "today only", "your account will be closed"
!
Authority abuse — claims from "the CEO", "the Governor", "Internal Audit"
!
Secrecy — "do not tell anyone", "this is confidential between you and me"
!
Look-alike domains — extra letters, dashes, country suffixes, .com vs .co
!
Generic greeting — "Dear customer", "Dear colleague" instead of your name
!
Off-channel request — work topic asked via WhatsApp, personal SMS, voicemail
!
Wire transfer / gift cards — never legitimate when received unsolicited
!
Trust your gut — if something feels off, it is. Report, don't click.

Module 04 · Integration Guide

Bringing PII filtering into your workflow

Below is a reference architecture for how Bank Al Etihad teams should embed PII filtering — from the open-source pii-filter library extended for our region.

3 PLACES TO PLUG IT IN
Where the filter sits
1. Email gateway — outbound DLP scanner intercepts emails with PAN, ID, or IBAN heading externally
2. AI / chatbot prompt layer — before any prompt is sent to an external LLM (ChatGPT, Claude, etc.), redact PII first
3. Support-ticket pipeline — incoming customer messages are auto-tagged and PII fields masked for analyst access
COMPLIANCE EVIDENCE
What auditors will ask
Logs of every redaction event, retention of original encrypted copies in a vault for 90 days, role-based access to unmasking, and quarterly accuracy testing. We provide all of these as part of the Etihad Insights package.
// reference snippet — npm install pii-filter
// Example usage adapted for banking context import { PIIFilter } from 'pii-filter'; // Extend with Jordan-specific patterns const filter = new PIIFilter({ locale: 'jo-ar', patterns: { iban: /JO\d{2}[A-Z]{4}\d{22}/g, nationalId: /\b\d{10}\b/g, phone: /\+?962[\s-]?7[789]\d{7}/g, pan: /\b(?:\d{4}[\s-]?){3}\d{4}\b/g } }); const result = filter.sanitize(message); console.log(result.cleaned); // safe to log console.log(result.entities); // audit trail

Note: HabaneroCake/pii-filter is archived (MIT licensed). For production banking use, pair its open-source patterns with a maintained engine (Microsoft Presidio, AWS Comprehend, or a self-hosted custom build) — Etihad Insights can advise.

Module 05 · Banking Essentials

The non-negotiable 10

If you remember nothing else from today, remember these. They are the daily behaviours that block 95% of attempted intrusions against banks.

✓
Lock your workstation every time you leave it. Win+L. No exceptions.
✓
Use the MFA app, not SMS. SIM swap attacks are real in Jordan.
✓
Verify wire transfer requests by callback to a known number — never the one in the email.
✓
Never reuse passwords across banking systems and personal accounts.
✓
Report phishing via the Outlook button — not by replying or forwarding.
✓
Encrypt removable media. No USB drives without BitLocker / approved policy.
✓
Clear desk policy — no customer documents visible at end of day.
✓
Verify identity before discussing any account, even with someone "from IT".
✓
Patch your machine within 24 hours of prompt. Don't snooze the update.
✓
If in doubt, escalate. Three minutes of SOC time beats a million-dinar incident.

Prepare

2
Detect

3
Contain

4
